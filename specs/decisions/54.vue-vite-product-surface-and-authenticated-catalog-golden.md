# ADR 54: Vue/Vite 产品面与 authenticated Catalog CRUD Golden

- Status: Accepted
- Implementation: Current-contract product vertical implemented
- Date: 2026-07-19
- Owners: `@prodivix/prodivix-compiler`、`@prodivix/runtime-core`、`apps/web`、Golden Conformance

## Context

ADR 48 证明了 Vue 3 + Vite 可以复用 canonical Data contract、standalone runtime、Executable Snapshot v6、
Browser/Remote provider 与独立工程 Gate，但第一纵切有意只接受 Data、Asset reference 与 project config，不能从
Blueprint 运行 PIR，也不能消费 Route/Auth/Server Runtime。S7/S8 因而仍缺一个由同一 canonical Workspace 驱动的
第二 target 产品旅程：受保护 Route、authenticated loader/action、Catalog CRUD、Binary Asset、Test、Remote snapshot 与
standalone export 必须在 Vue target 中共享 React 已冻结的领域契约，而不是另建 Vue 私有作者态或安全协议。

这次扩展仍属于 G2 的受控 target proof，不等于开放 G6 public Target SDK，也不要求把 React source 转译成 Vue source。

## Decision

### Vue compiler consumes canonical product contracts

当 Workspace 含 PIR document 时，Vue/Vite compiler 直接消费 canonical Workspace PIR、RouteManifest、Auth config、
Server Function code profile、Data Source、client-owned CodeArtifact 与 reference-only Asset：

- `prodivix-pir-runtime.ts` 使用 Vue `defineComponent`/`h` 投影 element、component instance、slot、Collection region、
  value binding、Data lifecycle 与 typed event dispatch；runtime value identity 保持 semantic symbol 与 PIR local symbol 的
  显式映射。
- `prodivix-workspace-app.ts` 拥有 route match/navigation、guard、loader、typed action 与 revalidation composition；Server
  Function 只经 source-free standalone adapter 调用。
- client-owned CodeArtifact 可进入目标工程；带 canonical Server Function profile 的源码不得进入 client files。
- Binary Asset 只经共享 exact-byte materialization planner 投影，digest、长度与 media type 验证规则不分叉。
- 每个生成模块保留 Workspace document/source trace；Vue runtime 不扫描其他编辑器内部状态。

没有 PIR document 的 ADR 48 Data explorer 继续兼容，避免破坏已有 controlled portability fixture。

### Product entrypoints

当前产品以同一 target identity `vue-vite` 提供三个显式入口：

1. Export Code 的 Vue/Vite 文件树、diagnostic、source preview 与 ZIP；
2. Workspace Test 的 React/Vite、Vue/Vite framework selector，两个 target 都维持强制 mock-only Data 与 deterministic
   authenticated Server Runtime fixture；
3. Blueprint Run Mode 的 framework selector，与 Browser/Remote provider selector 正交，不复制 Session/Job owner。

选择 target 只改变 compiler/executable projection。Execution Center、Network、Console、Terminal、Files、Test Report 与
SourceTrace 继续消费 provider-neutral Session contract。

### Auth/Server and export hard cut

- deterministic Workspace Test 可注入 metadata-only principal、permission、guard/loader/action fixture；fixture 只存在于
  Executable Snapshot provider materialization，不写 Workspace，也不进入 production export。
- Remote Preview 使用 execution-parent Data/Server gateway target，并传播 `network`、`environment-binding`、
  `data-stream`、`server-function` 的真实 capability requirement。
- 含受保护 Server Runtime 的 static Vue ZIP 必须以现有 `WKS-EXPORT-SERVER-GATEWAY-REQUIRED` 阻断；不得把
  authenticated action 降级为浏览器内 mock，也不得把 principal/session/Secret 或 server source 打进静态产物。
- Browser provider 不伪造 live Server Function capability；Browser 的 authenticated Golden 是 deterministic Test journey。

### Authenticated Catalog Golden

Living Golden Workspace 固定以下同一 revision 输入：

- “Authenticated Catalog” PIR page 与 data-bound product Collection；
- protected Route guard、authenticated loader 与 typed create/update/delete action；
- reference-only `/config/auth.json` 与 `workspace.owner` deterministic principal；
- list/get/create/update/delete Data operations、loading/empty/error/item regions；
- exact-byte PNG product Asset。

Gate 分两层：

1. contract matrix 验证 Vue Test snapshot 的 mock/auth provision、PIR/Route/Server/Asset files、server-source canary
   exclusion，以及 Remote Preview/Test/Build capability 和 strict codec round-trip；static protected export 必须 fail closed。
2. 独立临时工程执行 fresh install、`vue-tsc --noEmit`、Vitest、production build 与真实 Chrome smoke；浏览器必须看到
   authenticated loader value、成功解码 exact PNG，并完成 list/create/update/delete。

Remote 本纵切声明的是 exact snapshot/capability/codec closure，不把尚未执行的真实 Remote authenticated live Catalog
journey冒充通过证据。

## Failure semantics

- unsupported PIR shape、Route layout/outlet composition 或非当前 document domain 使用稳定 compiler diagnostic 阻断；
- guard/loader/action fixture 缺失、function ref drift、permission denial 或 adapter result drift fail closed；
- server source、Secret material、principal/session credential canary 命中任何 client file 即 Gate 失败；
- Asset materialization 缺失、digest/length/media drift 阻止 executable/export；
- target selector 变更会生成新的 immutable snapshot/request，不复用旧 target 的 active Job identity。

## Non-goals and remaining work

- public framework Target SDK、任意 Vue plugin/SSR/Nuxt 与 controlled visual/code round-trip 不属于本 ADR；
- Route layout chain/outlet composition、Design Token、NodeGraph 与 Animation Vue projection 继续稳定 fail closed；
- 真实 Remote authenticated Catalog live execution、permission-denial 产品旅程及跨 target Asset delivery/sanitize UI matrix仍需后续
  Gate；
- GitHub Actions 首次证据只能在后续提交推送后记录，workflow 配置不等于远端已通过。

## Consequences

- Vue 从 Data-only portability proof 扩展为当前 PIR/Route/Auth/Server/Asset contract 的可选择产品 target；
- React/Vue 共享 canonical owner、安全 Gate 与 provider-neutral execution contracts，没有产生第二套 Workspace 真相；
- G2 可以关闭“Vue current product surface + authenticated local Catalog Golden”子项，但不能据此关闭所有 Remote live、
  layout/outlet、Binary Asset 跨 target delivery 或 G6 SDK 缺口。
