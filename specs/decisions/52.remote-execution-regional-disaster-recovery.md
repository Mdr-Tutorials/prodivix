# ADR 52: Remote Execution regional PostgreSQL、Worker 与 traffic disaster recovery

- Status: Accepted
- Implementation: Implemented first vertical
- Date: 2026-07-19
- Owners: `@prodivix/runtime-remote`、`@prodivix/runtime-remote-postgres`、Remote Runner Control Plane

## Context

ADR 50/51 已分别解决同数据库的 Control Plane replica continuation，以及复制后的 Remote Terminal opaque row 由 related
AWS multi-Region key 在目标 Region 解密的问题。它们没有证明整个 Remote Execution authority 可以安全切换：execution、snapshot、
event、artifact、server authority、worker lease 与 Terminal row 可能尚未复制；旧区 HTTP/后台 sweep 也可能在 DNS 或入口切换期间继续
写入，形成双 active writer。PTY 是 Worker 进程资源，即使数据库与 KMS row 都可读取，也不能把已丢失进程伪装为已迁移。

本纵切需要同时固定三个边界：目标 PostgreSQL 必须是 exact checkpoint；流量切换必须先 drain 旧区且只产生一个 durable epoch；
Worker 继续必须遵守现有 lease/attempt fence，不自动 replay mutation 或复活 terminal Job。

## Decision

### Credential-free regional checkpoint

`@prodivix/runtime-remote` 定义 metadata-only `RemoteExecutionRegionalRecoveryCheckpoint` 与判定器；
`@prodivix/runtime-remote-postgres` 在单个 `REPEATABLE READ READ ONLY` transaction 内投影它。checkpoint 覆盖：

- immutable execution/request/provider/snapshot identity 与 exact status/cursor；
- snapshot blob/grant、ordered event、artifact descriptor 与重新校验后的 exact artifact bytes；
- cancellation、server authority、lease identity/attempt/time；lease token 只进入 SHA-256 state digest，不向 coordinator 暴露；
- Terminal session/revision/expiry 与 opaque sealed-state digest；
- bounded count/byte hard cut，损坏、缺 grant、cursor gap、artifact digest drift 或非法 lease shape 不形成 ready checkpoint。

目标 cursor/Terminal revision 小于源区时只返回 `wait-for-replication`；目标 ahead、同 cursor digest 不同、identity drift 与非法 lease
shape 都是 `blocked`。只有 exact state digest 才可继续。基础设施负责 PostgreSQL physical/logical replication 或 restore；生产代码不提供
application-level 跨区 row-copy API，测试中的双 schema copy 只模拟已经完成的基础设施 replication checkpoint。

### Worker and PTY recovery modes

exact checkpoint 根据 cutover 时刻得到唯一模式：

- `queued-claim`：尚未 claim 的 Job 在目标区正常首次 claim；
- `same-worker-continuation`：未过期 lease 的 worker id/token/attempt 不变，Worker 只切换 Control Plane origin，可继续 renew/ingest；
- `worker-reclaim`：lease 已过期且 attempt budget 尚未耗尽，由既有 PostgreSQL `claimNext` 原子产生 `attempt + 1`；旧 lease 被拒绝；
- `worker-recovery-exhausted`：不创建更多 attempt，既有 repository 路径以固定原因终止；
- terminal Job 只作为 terminal history 读取，不通过 DR 复活。

expired Worker 带有 Terminal 时，目标 broker 必须在 traffic exclusive section 中以 `transport-lost` 关闭旧 session、清空 token/mailbox，
再通过 exact revision sweep 删除 row。之后 attempt+1 才能创建新的 Terminal session id 和 PTY generation；旧 PTY 从不迁移。关闭/删除失败
则切换回滚，目标需重新复制 exact checkpoint 后重试。coordinator 不自动 replay mutation、stdin、Server Function 或 Data invocation。

### Shared traffic epoch and drain

所有启用 regional mode 的 Control Plane 共享一个独立 PostgreSQL traffic authority。每个 `/v1/**`、`/internal/v1/**` 请求以及
artifact/Terminal background sweep，在完整处理期间持有 deployment-scoped PostgreSQL advisory transaction shared lock，并确认本区是
current `active_region_id/epoch`。standby 固定返回 `region-standby`；authority 不可用固定返回 `region-unavailable`，不降级成本地写入。

cutover 取得同一 deployment 的 exclusive advisory transaction lock，因此先等待全部已接受请求 drain，并阻止任何区域获得新 permit；
随后在该 exclusive section 内重新 capture source/target、执行 exact recovery assessment、必要时撤销旧 Terminal，最后以
`expected epoch + source region` CAS 更新 active region。每次成功切换在同一 transaction 写 immutable cutover evidence：source/target、
new epoch、checkpoint digest 与时间。stale operator、重复命令或两个并发 operator 只能有一个成功。

`GET /healthz` 是不依赖 authority 的进程 liveness；`GET /readyz` 只有 current active region 返回 ready，供 load balancer 在外部 DNS/
Anycast/Global Accelerator 切换时 fail closed。该 contract 不实现 DNS vendor API，但证明应用写 authority 与入口 readiness 使用同一 epoch。

### Deployment configuration

regional mode 必须 all-or-none 配置：deployment id、local region id、initial active region id 与 shared traffic authority database URL。
traffic authority 必须是所有区域可达的单写一致性域，不能指向各自独立且异步复制的本地区表。执行数据库复制、RPO/RTO、backup restore、
网络入口和 Worker 调度仍由基础设施/runbook 拥有；应用 Gate 验证其结果与切换 fence。

## Failure semantics

- replication lag/drift、target ahead、traffic epoch conflict、Terminal revocation failure均保持旧 active epoch；
- target Terminal 已撤销但 epoch 更新失败时，旧源区仍安全 active，目标区必须重新复制再重试；不会尝试逆向合并；
- traffic database outage 使业务/readiness fail closed，但 `/healthz` 仍可用于进程诊断；
- old-region in-flight request 在 exclusive cutover 前完成；cutover 成功后新旧 Worker/Client 请求都必须重新取得目标区 permit；
- checkpoint、cutover evidence、HTTP error 均不包含 lease token、bearer、KMS ARN/ciphertext、Secret value 或 application input/output。

## Verification

- pure contract 覆盖 target missing/behind/ahead、same-cursor drift、queued/live/expired/exhausted/terminal mode；
- 真实 PostgreSQL 双 schema Gate 覆盖 repeatable-read checkpoint、cursor lag、state drift、shared request drain、epoch CAS、standby hard cut 与
  durable cutover evidence；
- 两个真实 HTTP Control Plane Gate 覆盖 standby liveness/readiness、live worker 同 lease 续租、旧区 transition hard cut；
- Worker-loss Gate 覆盖 expired attempt 1 -> attempt 2、旧 lease rejection、旧 Terminal row close+sweep 与新 session/PTY generation；
- GitHub `G2 PostgreSQL Gates` 使用 PostgreSQL 16 service 运行完整 regional DR drill。

## Non-goals and remaining work

- 不在应用内复制 PostgreSQL，不承诺某个云厂商的 DNS/Anycast、database replica promotion 或 Worker scheduler；
- 本地 Gate 不是跨真实 Region 的 RPO/RTO 测量；首次 cloud PostgreSQL/MRK/traffic live drill evidence 仍待取得；
- 多 execution/workspace 批量 checkpoint、operator UI、自动 runbook、长期 evidence retention/export 与 chaos cadence 继续建设。
