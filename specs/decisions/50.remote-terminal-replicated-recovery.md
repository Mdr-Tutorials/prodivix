# ADR 50: Remote Terminal encrypted replicated recovery

- Status: Accepted
- Implementation: Implemented first vertical
- Date: 2026-07-19
- Owners: `@prodivix/runtime-core`, `@prodivix/runtime-remote`,
  `@prodivix/runtime-remote-postgres`, Remote Runner Control Plane

## Context

ADR 40 已固定 Terminal 的 capability/grant/lease、stdin client sequence、output cursor、双预算、短期 access
token、Worker PTY 与跨 chunk Secret redaction。此前 Control Plane broker 仍以进程内 `Map` 保存会话：只要请求命中
同一副本，resume 可以旋转 token 并从 cursor 继续；副本退出后，未确认 stdin、output replay、幂等 fingerprint、
token digest 和 redactor pending suffix 会一起丢失。

把会话简单改成 sticky routing 只能隐藏副本切换，不能在 owner process 消失后恢复。把原始状态直接写入 PostgreSQL
又会使 raw stdin、fingerprint salt 与可能包含 Secret 前缀的 redactor suffix 暴露给数据库、备份和运维查询面。

## Decision

### Core checkpoint

`@prodivix/runtime-core` 提供 versioned `ExecutionTerminalCheckpoint`，只包含继续同一状态机所需的有界状态：

- exact terminal/execution/job/provider/capability identity；
- immutable budget、snapshot/revision/size/lease/output accounting；
- retained output records、salted input fingerprint tail 与 fingerprint salt；
- 不包含 adapter、listener、Promise 或 raw stdin。

恢复时严格校验 exact identity、字段集合、budget、cursor 连续性、byte accounting、timestamp/close shape 与 fingerprint
顺序。当前 grant 可以单调延长 checkpoint lease，不能缩短或漂移。续期后的 expiry 是状态机唯一有效 expiry；不再保留
旧 grant 的独立过期变量。

`ExecutionSecretTextStreamRedactor` 同时提供 versioned checkpoint。它只保存仍可能组成受保护值的 pending suffix，
并按当前 Secret set 验证该 suffix。该 suffix 可能是 Secret material 的一部分，因此只允许进入后述加密 envelope。

### Encrypted authority row

每个 execution 最多一个 `remote_execution_terminal_sessions` row。数据库可见字段只有：

- `execution_id`、`terminal_session_id`；
- 单调 `state_revision`；
- cleanup 用 `expires_at`；
- `sealed_state` opaque bytes。

plaintext state 还包含 principal subject、worker id/attempt/lease-token digest、provider、Core/redactor checkpoint、
worker-output fingerprint、access-token digest/expiry，以及有界未确认 command mailbox。raw access/worker token 不保存；raw
stdin 只存在于该加密 mailbox，ack 后从下一 revision 删除。

Control Plane 的 PRT1 compatibility path 使用 AES-256-GCM key ring。Envelope 记录 key id，nonce 每次随机生成；authenticated additional data 固定绑定
domain、key id、execution id、terminal session id、state revision 与 expiry。修改 ciphertext/expiry、换 session/execution/revision 或移除旧 key
都会 fail closed。部署轮换先加入新旧 key并切换 active id；新 revision 用 active key写入，旧 row 仍可读取并在下一次变更时
自然重封装。生产 managed PRT2 data-key envelope、PRT1 migration 与 MRK recovery 由 ADR 51 扩展。

### Replica transaction

client 与 worker 的每个操作都执行同一流程：

1. 按 exact execution/session 读取 row；
2. 以 row identity/revision 验证并解密；
3. strict decode 后重建 Core controller 与两个 stream redactor；
4. 从 execution repository 重新校验 active status、provider 与 exact worker lease/attempt/token digest；
5. 执行 write/resize/signal/close、worker ack/output 或 token rotation；
6. 生成新 checkpoint、用 `revision + 1` 重封装并 compare-and-swap；
7. CAS 竞争时有界重读和重放纯 broker mutation。

重试不会重复 PTY effect：broker mutation 只追加 command mailbox，由 Worker 在 effect 成功后 ack。相同 client sequence/input
在竞争后返回 `duplicate`，相同 sequence/different input 返回 `conflict`；相同 worker output id/payload 返回 `existing`，
payload drift 关闭 generation。CAS retry budget 耗尽返回 retryable unavailable，不使用 last-write-wins。

### Recovery and cleanup

- 任意 Control Plane 副本可用仍有效的 access token 继续 read/write，也可由 owner-authenticated resume 旋转 token。
- Worker 继续持有原 PTY，并可向任意副本轮询同一 mailbox/cursor；不迁移 PTY process。
- sweep 遇到 checkpoint lease expiry 时先重新读取 execution repository。若同一 lease 已续期，原子延长 checkpoint；若
  execution 终止或 worker generation 漂移，则关闭 state、清空 mailbox、revoke access 后按 exact revision 删除。
- 关闭、删除与并发 resume/output 使用同一 CAS fence；sweeper 不删除无法认证/解码的 row，留给 operator 审计。
- Terminal checkpoint 是短期恢复状态，不是 Canonical Workspace、Execution Job event、artifact 或验证证据。

## Security boundary

- PostgreSQL adapter 类型上只接受 `sealedState: Uint8Array`，不接触 plaintext schema。
- access token 与 worker lease token 只比较 digest；Web bearer 仍不进入 React state。
- known Secret、常见 credential 与 key-ring base64 值进入 output guard；stdout/stderr pending suffix 跨副本继续脱敏。
- ciphertext authentication failure 对外只投影固定 identity-conflict，不反射 key id、nonce、plaintext 或匹配位置。
- plaintext、sealed envelope、command、output fingerprint 与 sweep 都有独立上限；一个 corrupt row 不触发无界解析。

## Verification

- Core checkpoint round-trip、identity/budget drift、renewed lease 与 redactor suffix tests；
- 两个 broker 实例共享同一 store，并使用等价 key ring 的独立 cipher 实例，覆盖 cross-replica stdin、CAS duplicate/conflict、worker mailbox、output cursor、
  token rotation、split-canary、lease renewal 与 worker-loss purge；
- AES-GCM tamper/AAD/key rotation tests；
- PostgreSQL real integration 覆盖 opaque byte-exact read、concurrent CAS、expiry lookup 与 revision-fenced delete；
- existing strict HTTP client/worker terminal endpoints 回归。

## Non-goals and remaining work

- 浏览器内有界终端仿真产品面后续已由 ADR 53 完成；Browser provider 继续显式 unsupported。
- 本 ADR 不把 Terminal state 变成用户可恢复的长期 shell，也不允许 execution/worker generation 变化后复活 PTY。
- managed AWS KMS/MRK cryptographic portability 已由 ADR 51 完成 first vertical；跨 region PostgreSQL/traffic disaster-recovery drill、CloudHSM/第二 provider 与 operator quarantine/repair UI 继续作为后续 Gate。
