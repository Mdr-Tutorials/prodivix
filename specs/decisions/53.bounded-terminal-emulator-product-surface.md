# ADR 53: 有界 Terminal emulator 与 Execution Center 产品面

- Status: Accepted
- Implementation: Implemented first vertical
- Date: 2026-07-19
- Owners: `@prodivix/runtime-core`、`apps/web`

## Context

ADR 40/50/51/52 已经固定 Remote Terminal 的 grant、lease、PTY、stdin 幂等、output cursor、跨副本恢复、managed
KMS 与 regional recovery，但 Execution Center 仍把每个 output record 当成普通文本追加。该视图无法正确消费 ANSI
光标覆盖、清屏、颜色、alternate screen、application cursor key 或 bracketed paste；重连后的 replay 也没有一个明确的
screen state owner。直接把原始 ANSI 或第三方 terminal 私有状态塞进 React，会绕过 identity/cursor/byte budget、gap 与
credential redaction 边界，也会让 copy/accessibility 投影重新暴露控制序列或 SGR concealed 内容。

本纵切需要建立一个 DOM-free、transport-neutral、可丢弃且有界的 emulator projection。它不能成为第二套 PTY/Session
authority，不能把任意终端文本伪装成 SourceTrace，也不能给 Browser provider 增加伪造 shell fallback。

## Decision

### Runtime Core owns the disposable screen model

`@prodivix/runtime-core` 提供 `createExecutionTerminalEmulator`。输入只接受现有 strict
`ExecutionTerminalOutputRecord`；controller 在消费前校验 exact terminal/execution/job identity、递增 safe cursor、UTF-8
byte length 与 output chunk budget。重复 cursor 幂等忽略，cursor 跳跃投影固定 `[TRUNCATED]` gap marker。

parser state 只允许在同一 stdout/stderr stream 的连续安全 record 间延续。gap、`redacted`、`truncated` 或再次命中通用
credential redactor 的 record 在前后形成 hard parser boundary，不能把攻击者控制的半个 escape sequence 与下一块拼接。
control/OSC 序列均有 code-unit hard cut；异常序列被丢弃并计数，不回显、不响应。

screen、primary scrollback、alternate screen、cursor、style、title 与 metrics 都受固定 rows/columns/scrollback 预算约束；
resize 只改变该可丢弃模型，不修改 Terminal Session、Job event 或 Workspace。

### Frozen VT subset

first vertical 固定以下可测试子集：

- CR/LF/BS/TAB/BEL、宽字符与 combining mark；
- cursor absolute/relative、save/restore、scroll region、insert/delete/erase character/line/display；
- SGR reset、bold/dim/italic/underline/inverse/conceal/strike、16/256/RGB foreground/background；
- cursor visibility、wraparound、application cursor keys、bracketed paste 与 alternate screen；
- OSC 0/2 的 bounded/redacted title。

OSC 52 clipboard、OSC 8 hyperlink、DCS/APC/PM payload、device query response、Sixel/Kitty graphics 与任意 host integration
明确忽略。它们不是通过 unknown-sequence fallback 暗中启用的能力。

### Safe render, copy, and input

Execution Center 只渲染 emulator 的 immutable line/run projection，不渲染 raw record data。SGR concealed cell 在 public
line、screen-reader live region 与 copy 中投影为空格；copy 从 rendered scrollback/screen 重新执行 credential redaction 与
128 KiB hard cut，因此不包含 ANSI、OSC 或隐藏文本。

Web keyboard adapter 覆盖 printable、Enter/Backspace/Tab/Escape、cursor/navigation、F1-F12 与 Control key；
`Ctrl+C` 使用既有 execution-scoped interrupt signal，`Ctrl+Shift+C/V` 留给浏览器。paste 规范化 newline，可按 PTY mode
加入 bracketed paste，超过单次 16 KiB stdin budget 时整体拒绝而非静默截断。

快速键盘输入进入仅存在于 hook ref 的有界队列：最多 256 chunks/32 KiB，同一时刻只有队首是 unacknowledged input。
transport 丢失后保留 exact bytes/clientSequence，cursor/token resume 成功后只重试该 exact input；Core/Control Plane 的
fingerprint/idempotency contract 决定 `accepted | duplicate | conflict`。close、execution identity drift 与 unmount 会清空队列，
stdin 不进入 React state、Session history、artifact 或 Workspace。

## Failure semantics

- identity、cursor、byte-length 或 chunk budget drift 进入固定 `output-invalid` 状态，cursor 不前移，输出不渲染；
- unsupported/malformed control sequence 只增加 bounded metric，不执行 host action，也不向 PTY 自动写响应；
- local output/scrollback drop 显式保留 gap，不伪装为完整历史；
- paste/queue 超预算明确拒绝，不能截断成另一条 shell command；
- reconnect 只能延续同 terminal session；new session 会 reset screen、cursor、queue 与 emulator identity。

## Verification

- Runtime Core contract/property tests 覆盖跨 record ANSI、cursor/erase、SGR、alternate screen、scrollback/resize、宽字符、
  OSC hard cut、gap/redaction/truncation boundary、conceal/copy 与 fragmentation invariance；
- Web tests 覆盖 normal/application key、Control/AltGr、bounded bracketed paste、ANSI style render、rapid ordered input、
  exact unacknowledged retry、interrupt、accessible textbox/live region 与 invalid output fail-close；
- `pnpm run verify:g2:terminal-emulator` 是本地/CI 可重复 Gate，并接入 `G2 Execution Contract Matrix`。

## Non-goals and remaining work

- 不承诺完整 ECMA-48/xterm compatibility、terminal multiplexer、shell completion、search/selection、graphics 或 host clipboard；
- Browser ExecutionProvider 继续显式 unsupported，不提供本地伪 shell；
- emulator 是 presentation projection，不为终端文本发明 SourceTrace；runtime FS 采纳仍只走既有 diff/proposal/Transaction；
- 真实 cloud regional RPO/RTO、live MRK、operator repair UI 与更广泛 recovery evidence 仍由 ADR 51/52 后续 Gate 负责。
