## 背景

Animation 编辑器原有 Play 由 React 组件私有 `requestAnimationFrame` 循环驱动，只按单次 `durationMs` 取模。该路径没有执行 canonical delay、iterations、direction、fillMode 和 timeline easing，也没有 revision-bound Job、取消、timeout、SourceTrace 或共享 Console。

连续动画具有持续副作用。新 Job 启动时，旧 Job 可能仍有异步帧正在写入，因此只调用 cancel 并立即替换 Session 不足以保证新 revision 不被旧帧覆盖。

## 决策

### 三层 owner

```mermaid
flowchart LR
    Workspace[Canonical Animation document] --> Request[Animation ExecutionRequest]
    Request --> Provider[@prodivix/animation provider]
    Provider --> Playback[Domain playback lifecycle]
    Playback --> Port[Animation Runtime Port]
    Port --> Scheduler[Browser RAF scheduler]
    Port --> Lease[Generation-fenced effect lease]
    Lease --> Projection[CSS / SVG preview projection]
    Provider --> Job[ExecutionJob]
    Job --> Session[Execution Session]
    Session --> Center[Shared Execution Center]
```

- `@prodivix/animation` 拥有无 DOM scheduler/effect host port、完整 timeline 播放语义、same-context ExecutionProvider、领域 preflight、trace 和 diagnostic 映射。
- `@prodivix/runtime-browser` 只拥有 one-shot RAF scheduler、AnimationFrame 到 CSS/SVG 的浏览器投影，以及 React-neutral effect store；它不拥有 Workspace resolution 或 timeline lifecycle。
- `apps/web` 在 composition root 中绑定 exact Workspace snapshot、Animation document、浏览器 runtime port、稳定 Session 与编辑器 UI。

### Runtime Port

`AnimationRuntimePort` 由 one-shot `AnimationFrameScheduler` 和 `AnimationEffectHost` 组成。Scheduler 时间戳必须有限，回调只执行一次，取消必须幂等。Playback 使用 `scheduler.now()` 的基准差值，不累计 frame delta；上一帧 effect apply 完成后才调度下一帧，形成明确 backpressure。

Effect Host 声明 `style / css-filter / svg-filter` capability，并对 document-qualified target 做准备检查。每次 playback 获取独立 lease；RuntimeFrame 包含 document、timeline、binding、track contributor、elapsed/cursor 和中立 AnimationFrame。

### Generation lease

Browser effect store 为每次 acquire 分配递增 generation：

1. 新 lease 自动使旧 lease 失效并清空旧派生效果。
2. 旧 lease 的迟到 apply、release、cancel cleanup 全部变为 no-op。
3. reset、dispose 同样推进 generation。
4. completed 且 fillMode 为 forwards/both 时 retain 最终 frame；其他成功结果以及 cancelled、timed-out、failed 都 clear。

因此 Session 可以在旧 Job 尚未 terminal 时启动新 revision，而不会发生旧副作用覆盖新预览。

### Timeline 生命周期

正式 playback 复用领域 evaluator，统一执行 delay、finite/infinite iterations、normal/reverse/alternate direction、fillMode、keyframe easing 与 timeline easing。每帧结果只进入 effect host，不写 Workspace，也不进入 Job history。

Job 只发布 timeline root start/end trace、一次启动日志、diagnostic 与 terminal result；start/end 使用同一个 spanId。这样无限动画不会以 RAF 频率无限扩张 Session event history。CodeSlot custom easing、shader 和 timeline script 在对应 execution capability 尚未建立前 fail closed，不允许静默 succeeded。

### Revision-bound 产品入口

Animation 编辑器 Play 前先 flush 当前可逆 Command 链，再重新读取 Canonical Workspace，并以 exact snapshot identity 创建新的 Animation Job。Session identity 为 `workspace:{workspaceId}:animation:{documentId}`；Run/Stop/Restart 与 Console 复用共享 Execution Center。

静态 scrub 继续使用本地确定性 evaluator，但连续播放不再拥有 React 私有 RAF 或 `playing` 状态。运行帧 cursor 和浏览器 effect snapshot 是可丢弃的视图/运行态，不进入 Workspace、localStorage 或 Outbox。

旧 `buildAnimationPreviewSnapshot*` API 已删除。浏览器层只公开已经求值的 `AnimationFrame -> AnimationPreviewSnapshot` 投影，不保留第二套 lifecycle。

## Runtime Core 同步收敛

持续副作用暴露出两项公共不变量，现已同时修复：

- Provider registry 要求 `job.request` 保留 exact canonical request object，不能用部分字段相等的替代 request。
- 激活已完成 Job 时，Session 保留完整历史事件，但历史 state replay 不得让当前 terminal status 或 updatedAt 倒退。

## 当前能力边界

本纵切完成单 timeline、same-context、Browser PIR target 的正式预览执行。Route transition、timeline composition、reduced-motion policy、CodeSlot execution、GPU shader effect、Remote provider、性能采样和视觉回归证据继续在 G2/G3 收敛。
